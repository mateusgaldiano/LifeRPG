Operando localmente na sua máquina de arquivos, o Antigravity atuou como o executor prático. Ele lia o código-fonte gigante (4000+ linhas), aplicava edições limpas via patches e rodava comandos Git e Node diretamente no terminal do Windows.

Atuando na nuvem, o Claude serviu como arquiteto lógico. Ele projetava as fórmulas mais complexas (como as faixas do multiplicador de streak e os prompts de persona do Mentor Iroh) e exportava esse raciocínio conceitual para ser processado localmente.

💡 A Ponte do Google Drive: Programando em Qualquer Lugar

O Antigravity (Gemini) atuando localmente no PC possui a superpotência de alterar e criar arquivos de código diretamente. O Claude, pelo chat do navegador ou desktop, consegue ler perfeitamente esses arquivos copiados. Mas e se você quisesse programar na rua, longe do computador?

No celular, não existe aplicativo do Antigravity para rodar comandos locais, e o aplicativo do Claude Mobile também não possui qualquer acesso às pastas do seu PC. Para resolver essa limitação de mobilidade, criamos uma solução inteligente usando a computação em nuvem:

A pasta do seu PC local foi vinculada automaticamente ao seu Google Drive (Backup & Sincronização automática do próprio Drive, gerida nos bastidores com os rastros .tmp.drivedownload e .tmp.driveupload). Com isso, quando você estava fora de casa, bastava abrir o arquivo de código no celular pelo Google Drive para obter o código mais atualizado do app.

Você podia ler e editar o documento de onde quer que estivesse, colar o trecho atualizado no Claude Mobile, pedir ajustes no ônibus ou na rua, e salvar as alterações na nuvem. Ao chegar em casa e abrir o computador, a pasta já estava perfeitamente sincronizada com a version mais recente dos seus rascunhos, pronta para o Antigravity prosseguir com a codificação local e deploys automatizados no Git. A nuvem virou a ponte perfeita de portabilidade e sincronia!

🚨 O Registro Geral de Bugs e Incidentes

Durante o intercâmbio de código e deploys rápidos, enfrentamos e mitigamos diversos bugs. Veja o consolidado dos incidentes de desenvolvimento:

Incidente / Bug Severidade Esforço Cluster / Área Causa Raiz & Solução Aplicada
Emoji/Unicode Corrompido Crítico Pequeno Encoding / Git Transferência de arquivos fonte corrompeu bytes UTF-8 de emojis (como `🛏️` e `⚡`). Solucionado com script local `fix_encoding.js` e sanitizador automático nos métodos de load do LocalStorage.
Chave Ausente em switchTavernaTab Crítico Pequeno Sintaxe JS Durante merge da aba de skins, a IA apagou a chave `}` do bloco `else`. Travava o parse do arquivo todo (tela branca). Resolvido via terminal do Antigravity usando analisador de sintaxe (`node -c`).
Dungeon ativa + Nenhuma Missão Crítico Pequeno Interface / UX O fallback de "NENHUMA MISSÃO ATIVA" aparecia mesmo se houvesse uma Dungeon roxo-neon em andamento. Corrigido adicionando verificação lógica no condicional do renderer.
Header Mobile Desalinhado Médio Pequeno Layout CSS O cabeçalho `.brand-header` com `sticky` sumia em determinadas rolagens do mobile. Alterado para `position: fixed` no CSS com body scroll e compensação de padding.
Bloqueio de CORS na API Claude Médio Médio IA / Chamada API Requisições diretas de chat eram barradas pelas políticas do navegador. Criado campos seguros para o usuário definir chave de API e Proxy URL customizada nas Configurações.
Botões Paravam de Funcionar Médio Pequeno Interface / DOM Regressões recorrentes onde botões de check de quests, exclusão e abas funcionavam na primeira carga, mas paravam de responder após certas atualizações. Causa: a re-renderização limpava elementos usando innerHTML, destruindo os event listeners. Resolvido com delegação de eventos dinâmica no container pai ou reinicializando os listeners em cada render.
Cabeçalho/Rodapé Solto (iOS Safari) Médio Médio Layout / CSS No iOS, o cabeçalho e o rodapé cobriam inputs ou sumiam devido às barras dinâmicas do navegador Safari. Resolvido ajustando a altura da viewport usando a unidade flexível 100dvh (dynamic viewport height) no CSS, garantindo que cabeçalhos e rodapés fiquem fixados sem cobrir os dados.
Falha no Reset do App Crítico Pequeno LocalStorage / Boot O botão de reset total travava o app ou mantinha resquícios no localStorage, gerando erros de carregamento na reinicialização. Solucionado limpando explicitamente todos os dados com localStorage.clear(), inicializando imediatamente o estado template vazio padrão no cache e recarregando a página de forma limpa.
Botões que Sumiam com a Tela (White Screen) Crítico Médio JS Execution Ao tentar excluir uma quest ou abrir modais com dados nulos, estourava um erro Cannot read properties of null que quebrava o loop do JavaScript, fazendo a tela inteira sumir. Resolvido com programação defensiva (blocos try/catch e validações preliminares if (!data) return;) para evitar falhas catastróficas.
Atividades Sumindo Sozinhas Crítico Médio Filtros / Datas Atividades ativas desapareciam da lista sem intervenção. O erro era o uso de data local com formatos estáticos, que causava perda de sincronia após a meia-noite em fusos não-UTC. O filtro de renderização interpretava a data como concluída/expirada. Resolvido padronizando datas com o fuso do navegador usando a string formatada localDateStr().
Atividades Duplicadas (Clones) Médio Pequeno Mecânica / CRUD Duplo clique rápido no botão de salvar adicionava a mesma quest duas vezes no array com IDs idênticos, quebrando os cálculos de XP. Resolvido com desabilitação temporária do botão após o primeiro clique (debouncing) e geração robusta de UUIDs únicos para cada quest.

Abaixo, detalhamos a autópsia técnica de dois bugs que serviram como grandes lições:

👾 Autópsia A: A Guerra das Codificações (Encoding UTF-8 vs ANSI) ▼

Sintoma: Emojis fundamentais (como a cama `🛏️` e o raio `⚡`) viraram caracteres corrompidos estranhos (Mojibake) durante transferências de buffers de rede e Drive.

Solução: Criamos scripts em Javascript como `fix_encoding.js` no scratch para varrer e reescrever o arquivo. No código de produção, colocamos um detector que filtra e sanitiza dados antigos corrompidos no `localStorage` ao carregar a página.

app.js — Sanitizador de SavesJS
// Sanitiza saves corrompidos por falhas de encoding entre IAs
function cleanObjectEncoding(obj) {
-    return JSON.parse(obj); // Quebrava se houvesse emoji quebrado
+    let raw = JSON.stringify(obj);
+    raw = raw.replace(/<\\u0005/g, '🛏️').replace(/=%/g, '⚡');
+    return JSON.parse(raw);
}
🔑 Autópsia B: O Caso da Chave Desaparecida (Syntax Errors) ▼

Sintoma: Após mesclar o menu de abas da Taverna, a tela do app ficou totalmente em branco com o erro de console SyntaxError: Unexpected end of input.

Solução: A IA comeu a chave } de fechamento do bloco de condições. A resolução foi rodar a análise de sintaxe nativa do Node diretamente no terminal do Antigravity antes do commit, apontando a linha exata sem a chave:

Terminal PowerShellbash
$ node -c app.js
C:\liferpg\app.js:4287
SyntaxError: Unexpected end of input
# Feito a correção da chave de fechamento ausente no switchTavernaTab!

🐙 A Evolução do Workflow: Do Upload Manual ao Git Automatizado

No início do desenvolvimento, o workflow apresentava bastante atrito humano. O Antigravity realizava as edições e exibia instruções textuais pedindo para que você realizasse manualmente o upload dos arquivos para o GitHub no seu computador.

💡 O Aprendizado de Capacidade do Terminal

Em uma das interações, você percebeu em meus logs de sistema que havia uma ferramenta de terminal chamada **`run_command`** e sinalizações de erro do tipo "git fail" quando o git local não encontrava os remotes no início da sincronização.

Curioso, você me perguntou no chat quais eram as minhas capacidades reais em rodar comandos na sua máquina. Ao explicar que eu podia executar comandos PowerShell diretamente sob a sua aprovação, você pediu: "A partir de agora, integre tudo de forma automática. Faça os commits e pushes você mesmo."

Este momento transformou nosso workflow. Nós aprendemos juntos a usar as capacidades do Git diretamente integradas na IA:

  • Commits estruturados: O Antigravity passou a fazer o stage (`git add .`) e commitar com mensagens de padrão semântico detalhando as melhorias aplicadas.
  • Gerenciamento de Remotes: Criamos o branch `dev` enviando deploys rápidos para o remote `dev-origin` (GitHub Pages de testes), e controlamos o branch `main` com remessa direta para o `origin` (deploy de produção).
  • Velocidade de Entrega: Em vez de gastar minutos arrastando arquivos para o GitHub, os pushes de testes passaram a ser concluídos em menos de 10 segundos, direto do chat da IA para a web!
Pipeline de Deploy automatizado pela IAGit PowerShell
# Fluxo executado autonomamente pelo Antigravity local a seu pedido
$ git add app.js index.html styles.css
$ git commit -m "feat: select habit type (daily/side) and add remove button to quest cards"
$ git push dev-origin dev:main
# Pushed to dev remote - Deploy ativo e testável no Pages em 5s!
