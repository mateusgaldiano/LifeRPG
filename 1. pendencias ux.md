# Pendências UX — LifeRPG OS v2.0
> Auditoria realizada via code review (app.js + lógica de jogo completa)
> Perspectiva: PM de Produto / UX Mobile Android

---

## 🔴 Crítico

### 1. `updateUI()` ausente após perks em `checkAllDailies()`
Os perks **Mente de Diamante** (+10 XP) e **O Sistema** (+1 Skill XP) são aplicados diretamente no `gameState` dentro de `checkAllDailies()`, mas `updateUI()` não é chamado ali — depende do caller (`toggleQuest`, `adjustWater`). Resultado: o XP dos perks só aparece visualmente no próximo clique do usuário, gerando lag visual perceptível.

**Fix:** Adicionar `updateUI()` ao final do bloco `if (allDone)` em `checkAllDailies()`, após `saveGameData()`.

---

### 2. Títulos do Avatar Zoom desalinhados com o sistema de Rank (Item 1 pendente)
A função `openAvatarZoom()` usa thresholds de nível próprios para os títulos:

```
LV5+  → "Aventureiro"
LV10+ → "Guerreiro de Elite"
LV15+ → "Herói Lendário"
LV20+ → "Semideus do Foco"
```

Mas o sistema de rank real (`RANK_THRESHOLDS`) vai até LV30 (RANK S). O título "Semideus do Foco" aparece no RANK A (LV20), e não existe nenhum título para RANK S (LV30+). Inconsistência de progressão — o jogador de RANK S não tem um título condizente.

**Fix:** Alinhar títulos do Avatar com os ranks reais (E/D/C/B/A/S) via `getRankForLevel()`. Este é o **Item 1 da fila de patches** confirmado como não implementado pelo Antigravity.

---

### 3. Bug: Dungeon ativa + "NENHUMA MISSÃO ATIVA" aparecem simultaneamente
Quando uma dungeon está ativa mas não há side quests manuais, o card da dungeon é renderizado no `sideContainer`, mas logo abaixo aparece o texto `NENHUMA MISSÃO ATIVA` — porque o check `if (gameState.sideQuests.length === 0)` não considera a dungeon ativa.

**Fix:** No patch do Item 9 (Dungeons), ajustar a condição do fallback para:
```javascript
if (gameState.sideQuests.length === 0 && !gameState.activeDungeon) {
    sideContainer.innerHTML = `<div ...>NENHUMA MISSÃO ATIVA</div>`;
}
```

---

### 4. `alert()` nativo na Taverna quebra a experiência imersiva
```javascript
alert(`Sucesso! Você liberou: "${reward.title}"`);
```
Em mobile Android o `alert()` nativo é feio, bloqueia o thread de renderização e rompe completamente a estética do app. O sistema já tem overlays customizados (Level Up, Penalty) — a Taverna deveria usar o mesmo padrão.

**Fix:** Criar um overlay de confirmação de compra reutilizável (similar ao `#level-up-overlay`) e substituir o `alert()` em `buyReward()`.

---

## 🟡 Médio

### 5. Boss Quest sem UI persistente na tela principal
A Boss Quest ativa só é comunicada via mensagem do Iroh no chat. Não há nenhum elemento visual fixo na tela mostrando qual Boss Quest está ativa e qual o progresso atual. Usuários que não leem o chat não sabem que ela existe.

**Fix:** Adicionar um card de Boss Quest no topo das Side Quests (semelhante ao card de Dungeon), com título, descrição e barra de progresso. Renderizar via `renderQuests()` quando `gameState.bossQuest && !gameState.bossQuest.completed`.

---

### 6. Streak display com risco de overflow no HUD mobile
```javascript
streakEl.innerText = `${gameState.streak}${multStr}${shieldStr}`;
// Ex com streak 30 + sinergia + 4 escudos: "30 · x1.15  🛡️🛡️🛡️🛡️◈"
```
Com streak alto + multiplicador + escudos, o texto fica longo demais para a linha do HUD num celular de 375px de largura. Risco de quebra de layout ou truncamento.

**Fix:** Limitar o display de escudos a ícones menores ou colocar o multiplier em linha separada (subscript abaixo do streak). Alternativamente: `white-space: nowrap; overflow: hidden; text-overflow: ellipsis;` no elemento.

---

### 7. Água (q-agua) sem barra de progresso visual
O contador `3/8 copos` é exibido como texto puro. Em um card escuro com tipografia pequena, é difícil de processar rapidamente como dado de progresso.

**Fix:** Adicionar uma mini barra de progresso horizontal abaixo do contador, preenchendo proporcionalmente (`current / target * 100%`), usando a mesma estética das barras de XP.

---

### 8. Sem indicador visual persistente de "dia completado"
Quando todas as dailies são concluídas, o feedback é apenas uma mensagem no chat. Não há nenhum estado visual permanente na aba Missões (ex: badge verde, checkmark, cor diferente no título "Missões Diárias").

**Fix:** Ao detectar `allDone` em `checkAllDailies()`, adicionar uma classe CSS ao header da seção de dailies (ex: `.all-complete`) com estilo distinto — cor verde/dourada, ícone de check — que persiste até o próximo reset diário.

---

## 🟢 Baixo / Polimento

### 9. Perks e Sinergias sem separação visual clara
Os badges de Rank Perks e Sinergias aparecem em sequência direta. Para um jogador de LV15+ com múltiplas sinergias ativas + 3 perks, a área vira uma massa de badges sem hierarquia.

**Fix:** Adicionar um label/separador entre os dois grupos (ex: `"PERKS"` e `"SINERGIAS"` em texto micro acima de cada grupo), ou separar por cor de borda (cyan para sinergias, dourado para perks).
